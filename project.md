이 기획의 목표는 **"최소 기능 제품(MVP)"**으로, 이름 입력 후 바로 맵에 접속해 다른 사람을 만나는 것에 초점을 맞춥니다.

---

## 📋 웹 기반 멀티플레이 '동물의 숲' 기획서 (MVP)

### 1. 프로젝트 개요

- **서비스명:** Human Crossing
- **핵심 가치:** 설치 없이 브라우저에서 즐기는 3D 커뮤니티 공간
- **주요 기능:** 닉네임 설정, 3D 월드 접속, 캐릭터 이동, 실시간 유저 동기화

---

### 2. 기술 스택 (JS 기반 최적화 조합)

프론트엔드 개발자에게 가장 익숙하면서도 강력한 도구들입니다.

| 구분          | 기술                        | 선택 이유                                          |
| ------------- | --------------------------- | -------------------------------------------------- |
| **Frontend**  | **React (Vite)**            | 컴포넌트 기반 UI 관리 및 빠른 빌드 속도            |
| **3D Engine** | **React Three Fiber (R3F)** | Three.js를 React 방식으로 선언적으로 사용          |
| **Physics**   | **@react-three/rapier**     | 캐릭터 충돌 및 지형 물리 계산                      |
| **Character** | **Ecctrl**                  | 상하좌우 이동 및 카메라 컨트롤러 라이브러리        |
| **Backend**   | **Node.js + Colyseus**      | 게임 상태 동기화 특화 웹소켓 프레임워크            |
| **Database**  | **MongoDB (Mongoose)**      | 유저 정보 및 맵 데이터를 JSON 형태로 유연하게 저장 |

---

### 3. 유저 시나리오 (Flow)

1. **진입 (Entry):** 메인 홈페이지 접속.
2. **로그인 (Intro):** 텍스트 필드에 '닉네임' 입력 후 [시작하기] 버튼 클릭.
3. **월드 로딩:** 3D 에셋(맵, 캐릭터 모델) 로드 및 서버(Colyseus) 연결.
4. **플레이 (Play):** \* `WASD` 혹은 방향키로 이동., 모바일도 지원 가능해야 함

- 다른 유저가 접속하면 실시간으로 맵에 나타남.
- 다른 유저가 이동하면 내 화면에서도 부드럽게 움직임.

---

### 4. 시스템 구조도

- **Client (React):** UI(닉네임 입력창) + 3D View(R3F). 서버에서 오는 '유저 리스트' 데이터를 `map` 돌려서 캐릭터 컴포넌트들을 렌더링함.
- **Server (Colyseus):** \* `Room`: 마을 공간 관리.
- `State`: 현재 접속한 모든 유저의 `{ id, name, x, y, z, action }` 저장.

- **DB (MongoDB):** 닉네임과 최종 위치 정보를 저장하여 재접속 시 유지.

---

### 5. 상세 구현 계획 (개발 우선순위)

#### **Step 1: 닉네임 입력 UI (React)**

- 단순한 `<input>`과 스테이트 관리.
- 시작 버튼 클릭 시 `isGameStarted` 상태를 `true`로 바꿔 3D Canvas를 띄움.

#### **Step 2: 3D 환경 구축 (R3F + Ecctrl)**

- 바닥(Plane)과 조명 설정.
- AI(Meshy 등)로 생성한 캐릭터 `.glb` 파일 로드.
- `Ecctrl`로 캐릭터 조작 확인 (로컬 환경).

#### **Step 3: 멀티플레이 연동 (Colyseus)**

- 유저가 접속하면 서버 `state.players`에 새 유저 추가.
- 클라이언트의 `useFrame` 안에서 내 캐릭터의 위치를 서버로 전송.
- 서버의 상태 변화를 구독하여 다른 플레이어 모델들의 위치를 업데이트.

---

### 6. 향후 구현 계획 (Post-MVP)

MVP 이후 단계적으로 추가할 기능들입니다.

#### **Phase 1: 비밀번호 로그인**

- 현재 닉네임만으로 접속하는 방식에서, **숫자 비밀번호(PIN)**를 추가하여 본인 확인 가능하도록 변경.
- 신규 가입 시 4~6자리 숫자 PIN 설정.
- 재접속 시 닉네임 + PIN 입력으로 인증.
- DB `players` 테이블에 `pin` 컬럼 추가 (해시 저장).

#### **Phase 2: 방명록 시스템**

- 월드 내 특정 오브젝트(게시판, 우체통 등)에 다가가면 **방명록 작성 UI**가 열림.
- 다른 유저가 남긴 방명록도 읽을 수 있음.
- 데이터 구조:
  - `guestbook` 테이블: `id`, `authorId`, `content`, `createdAt`
- API:
  - `POST /api/guestbook` — 방명록 작성
  - `GET /api/guestbook` — 방명록 목록 조회

#### **Phase 3: 미션 시스템**

- 월드 내 곳곳에 미션 포인트를 배치하여 유저가 수행할 수 있도록 함.
- 미션 예시: 특정 장소 방문, 오브젝트 상호작용, 다른 유저와 만남 등.
- 각 유저별 미션 수행 진척도를 서버에 저장.
- 데이터 구조:
  - `missions` 테이블: `id`, `title`, `description`, `type`
  - `player_missions` 테이블: `id`, `playerId`, `missionId`, `status`, `completedAt`
- API:
  - `GET /api/missions` — 전체 미션 목록
  - `GET /api/players/:id/missions` — 유저별 미션 진척도
  - `POST /api/players/:id/missions/:missionId/complete` — 미션 완료 처리

#### **Phase 4: 마이페이지 / 활동 기록 화면**

- 유저가 자신의 활동 기록을 한눈에 볼 수 있는 화면.
- 표시 항목:
  - 내가 작성한 **방명록** 목록
  - **미션** 수행 진척도 (완료/미완료)
  - 가입일, 마지막 접속일, 총 접속 횟수 등 기본 정보
- 다른 유저의 프로필도 조회 가능 (공개 범위 내에서).
